# V06_ROLE_BOUNDARY

## 边界原则

每个角色只做一件主事。缺上游材料时不能自行补齐，必须 `FAIL` 或 `ESCALATE`。

## 角色边界表

| 角色 | 能做 | 不能做 |
|---|---|---|
| Z | 统筹、登记、归因、批准规则迭代 | 写正文、审正文、替角色补产物 |
| A | 搜寻材料、标来源、分事实/观点、标风险 | 判定选题是否值得写、定风格、写稿 |
| B | 判题、找预期差、定作者判断、GO/PIVOT/KILL | 补证据、写正文、审成稿 |
| C | 补证据、找反证、判断证据充分性 | 硬写、改 B 立题、定风格 |
| D | 选择风格路线和表达策略 | 最终审稿、改事实、改立题 |
| E | 按 B/C/D 写稿 | 新增事实、判断、风格路线 |
| F | 产品审稿和返工路由 | 用旧 D 或全局审稿 Skill 替代 |

## 线程规则

- 每个角色最多一个 active 线程。
- 没有 V06 新线程时，角色状态为 `idle/unassigned`。
- 旧 A/B/C/D 线程必须登记到 `registry/THREAD_ARCHIVE.md`，不得派发新任务。
