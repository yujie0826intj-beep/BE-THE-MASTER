# V06_TOKEN_NOISE_CONTROL

## 目标

减少刷屏、重复读旧线程和无效重写，把 token 花在选题、证据和产品判断上。

## 冻结规则

1. 不启动 heartbeat 自动巡检。
2. 不继续 V05/R6 正文小修。
3. 不读取旧线程大段上下文。
4. 历史失败只读本地私密记录或已脱敏公开摘要，不读取旧线程大段上下文。
5. 每轮只允许一个明确状态动作。

## 低噪音执行

- 状态检查不通知，除非出现阻断或需要用户决策。
- 旧线程不复活，旧目录不写入。
- E 只允许一次修改，防止 C/D 式无限循环重演。
- F 审稿必须短而硬：先判产品，再解释原因。

## 高噪音信号

- 同一角色反复全量重读规则。
- 多轮只修禁词、不回到选题或证据。
- D/F 写长审稿卡但没有产品结论。
- Z 连续加门禁却不停止流程。

## Z 处理

出现高噪音信号时，Z 必须停止派发，给用户 2-3 个业务选项。
